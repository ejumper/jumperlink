Overview

An administrator account is a default account used in all versions of Windows OS. It provides full access to the resources and the configuration of those resources for a device, which in this case is a domain controller. Therefore, before beginning to do anything on a domain controller, one must be configured. It's worth specifying that an admin account on a domain controller necessarily gains access to resources in that domain and is therefore a target. It is paramount that the account properties are set up securely. This includes setting especially strong passwords and restricting Remote Desktop access through Group Policies. In this post I will walk through this process and as part of it I will also be going through the process of creating Organizational Units and users accounts.

Creating an Admin OU

I begin going to Server Manager > Tools > Active Directory Users and Computers.

In the left-hand navigation Panel I right-click on domain (in my case jumperdomain.net)

Then I go New > Organizational unit to create an OU to house the admin accounts.

type in the name and for my purposes I meant to uncheck the "Protect container from accidental deletion" but I forgot which isn't a bad thing because now I can figure out how to delete OUs that are protected from deletion.

after some scrolling through stack overflow I select the view tab > advanced features > right-clicked the OU > select Properties > go to Object & unselect the Protect object from accidental deletion

Creating an Admin User

Back on track I will now create a user account inside the _Admin OU by right clicking on the OU > selecting New > User

I configure my name since this is my Admin account. for the logon name I found the naming convention "a-ejumper" online and so I went with it.

After hitting Next I configure my password and for the purposes of the lab turn off "User must change password at next logon", hit Next, review the information & hit Finish

Now in order to give the account admin privileges I right click on it > select properties and in the properties panel select the Member of tab and in the Enter the object names to select dialogue box I enter Domain Admins, select Check Names, then OK > Apply > OK

I do a quick Window+L to log out (forgetting that I am in a VM and will also log out of my host OS, oh well)

Now I select Other users and sign in with my admin user account
