Overview

In this post I will outline my experience creating Virtual Machines using Oracle's VirtualBox. I have done this with several different OSs including Windows Server 2019 & 2025, Ubuntu Server 24, Windows 11 & Linux Mint, however, for the purposes of demonstration I will walk through the process of creating and configuring Windows Server 2019.

As this will not be my first time installing Windows Server 2019 as a VM I will take the time throughout to note different problems I had in past installations & how I addressed those problems.

Setup

Downloading Assets

on virtualbox.org

I downloaded hosts for my applicable host OS (Windows 11)

Then I downloaded the VirtualBox extension pack

Not strictly necessary but useful as I wanted to practice using VRDP

on microsoft.com

I download Windows 11

I download Windows Server 2019

Installing VirtualBox

I went through the Setup Wizard and installed VirtualBox, it is straight forward and doesn't require any configuring beyond whether you want a desktop shortcut or not.

In VirtualBox Manager

I selected New to begin setting up the Windows Server 2019 VM

I did some research on naming conventions for Virtual Machines and found this website which outlined that a VM name should identify what the device is, its location and its purpose

I figured since there was no question where the server was located I'd drop that and simply identify the device (WS19) and its purpose as a domain controller (DC) giving me the name WS19-DC-ADLAB

I then selected the .iso image I had downloaded and turned off unattended install so that I could manually configure the hardware,

on the next page I allocated 8GBs of RAM and 4 logical processors

my machine has 32GBs of RAM and based on my research the consensus off of various forum posts seemed to say that 8GBs of RAM would provide a smooth experience and leave me with plenty of room in case I decide to experiment with additional services or roles as well while also leaving enough resources to run the windows 11 VM and my host machine.

As with the RAM the 4 vCPUs I allocated out of my 16 total logical processors is well enough over the minimum requirements that it should easily allow the Server to run smoothly and retain enough resources for the windows 11 VM and my host machine.

for virtual hard disk space (after spending some time cleaning up my C: drive) I had 210GBs of free space so I allocated 80GB of disk space for the server making sure to leave the "pre-allocate Full size" check box off so that it would only use as much as it needed up to 80GB. I came up with this amount based on a rough estimate of the space I would need based on what I planned to use the VM for plus around 20GB to leave a healthy margin of error.

I originally installed the VMs on my much more empty D: HDD with more space, however this proved quite slow and it was well worth taking the time to free up space on my C: SSD as I more then made up the time spent in VM boot & OS startup alone

before launching the server I went into settings to add a few features I wanted.

under General > Advanced Tab

I changed the shared Clipboard & Drag'n'Drop settings to Bidirectional to make make file transfer between the host OS and my VM a little simpler

I happened upon this setting in when reading through the VirtualBox documentation after I had already done this lab a few times and it is a huge convenience. A testament to taking the time to look through the documentation before using new software

With everything configured. I can now select the WS19-DC on the left side panel under Tools and begin running the VM.

In the Windows Server 2019 VM

I verified the language & keyboard settings & selected next > Install now to begin the Setup wizard.

In the Setup Wizard

I selected Standard Evaluation since I won't be using any of the Datacenter Editions features & (Desktop Experience) in order to have access to the GUI. Then I accepted the license terms.

I then select the Custom installation and progressed through the wizard until in began installing Windows.

With the server now Installed I create an Administrator Password and hit finish.

Now on the lock screen there is one final hurdle for the server as the Ctrl+Alt+Delete that should unlock the screen does not work.

I says press Ctrl+Alt+Delete to unlock, however, this triggers the host device security screen instead.

the solution is to go to Input > Keyboard > Ctrl-Alt-Delete and you will be presented with the password prompt to unlock

Now inside the OS I make some minor configuration changes to help the server run more smoothly in the VM

Devices > Insert Guest Additions CD image...

then in file explorer the first thing I always do as a matter of preference

view > show/hide > check "file name extensions" & "hidden items"

then more importantly I navigate to This PC > CD Drive (D:) VirtualBox Guest Additions

select VBoxWindowsAdditions-amd64.exe

this installs a suite of drivers & features within the VM that will be helpful

allows you to create shared folders between the host and VM, improves performance a little, makes the mouse work better & allows you to resize the screen.

run through the wizard and hit Install > Finish (I select I want to manually reboot later since the next step will involve restarting)

Updates

Now I will perform the pending updates to the system. The quickest way to do this is to simply click the update icon on the right of the taskbar and click install now to install all the pending updates and once they're installed I restart again.

Renaming PC

While not strictly Necessary for my own convenience & to keep everything organized I will rename the PC

in Settings > About > Rename this PC I give it the same name as the VM (WS19-DC)

after selecting Next I restart the PC (if I had thought about it I could have done this before the updates and avoided the extra restart).
